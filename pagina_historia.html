História da Linguagem C

A linguagem de programação C foi criada em 1972 por Dennis Ritchie em “Bell Telephone Laboratories”, e foi usada por Dennis e seu colega Kenneth Lane Thompson para desenvolver o sistema operacional Unix, que acabou dando origem ao Linux, IOS e Android.

Originalmente, Unix foi desenvolvido em linguagem assembly, porém a linguagem C se tornou poderosa o suficiente para que a maior parte do kernel do Unix fosse reescrito em C, com a adição de características como o tipo struct. Esse foi um dos primeiros núcleos de sistemas operacionais implementados numa linguagem que não é assembly.

O objetivo principal foi evitar usar uma linguagem de montagem, ou “assembly”, que é próxima a linguagem de máquina, logo foi criada uma linguagem de alto nível, aos padrões da época. Com a divulgação, ela ficou popular entre muitos programadores, em especial graças ao livro de Brian Kernighan e Dennis Ritchie, “The C Programming Language” em 1978, que serviu como guia.

Por conta deste livro, originalmente a linguagem era conhecida pelo nome “C Kernighan and Ritchie”, ou “C K&R”. Esse nome agora serve para especificar esta antiga versão da linguagem C. A linguagem C K&R introduziu caracteristicas como biblioteca padrão de entrada e saída, tipos de dados “struct”, “long int”, “unsigned int” e algumas alterações de operadores.

Em 1983, a American National Standards Institute, ou ANSI, a equivalente da ABNT nos Estados Unidos, formou um comitê, X3J11, para formalizar o padrão da linguagem C, conhecido como ANSI C, que deveria ser seguido por todos os computadores da linguagem C, ele foi completo em 1989 e ratificado como ANSI X3.159-1989 "Programming Language C", conhecido também como C89.

Após isso, uma segunda edição do livro foi lançada em 1988, cobrindo ambos os padrões, o C K&R, e o ANSI C. Nos anos seguintes dessas publicações, algumas características “não oficiais” foram adicionadas à linguagem, suportadas por alguns compiladores, coisas que são utilizadas até hoje, como funções “void”, funções que retornam tipos “struct”, entre outras.

Nos finais da década de 1970, a linguagem C começou a substituir a linguagem BASIC como a linguagem de programação de microcomputadores mais usada, já na década seguinte, foi adaptada para uso no PC IBM, e sua popularidade começou a aumentar significativamente. Ao mesmo tempo, Bjarne Stroustrup, juntamente com outros nos Laboratórios Bell, começou a trabalhar no que seria a linguagem C++.

Em 1990, o padrão ANSI C, após sofrer modificações, foi adotado pela Organização Internacional para Padronização (ISO), que cria o grupo WG14 para cuidar da especificação da linguagem, assim foi renomeada para ISO/IEC 9899:1990, e ficou conhecida como C90.

Um dos objetivos das padronizações foi o de produzir um subconjunto do K&R C que incorporava muitas características não oficiais introduzidas subsequentemente, porém muitos programas não compilavam em certas plataformas ou em compiladores específicos, devido ao uso de bibliotecas e funções não padronizadas, e o fato de alguns compiladores não haverem aderido aos padrões.

Após o processo de padronização ANSI, as especificações da linguagem C permaneceram relativamente sem mudanças por algum tempo enquanto a linguagem C++ continuava evoluindo.

Em 1999, foi publicada a norma ISO 9899:1999, normalmente referido como C99, e adotado como padrão ANSI em março de 2000, ela inclui características semelhantes à linguagem C++, como funções em linha, adição de vários tipos de dados novos, suporte para comentários e bibliotecas novas.

Em 2007, iniciou uma antecipação de outra revisão de C, informalmente chamada de C11 ou C1X, publicada em dezembro de 2011 como ISO/IEC 9899:2011, o comitê dos padrões C adotou regras para limitar novos recursos que não tenham sido testados.

Em 2017, começou o desenvolvimento de uma nova revisão para substituir a C11, conhecida como C17 ou C18. Ela visou melhorar a revisão anterior sem introduzir novas funcionalidades, ela foi publicada em junho de 2018 como ISO/IEC 9899:2018.

No momento está sendo desenvolvida a versão ISO/IEC 9899:2024, conhecida informalmente por C2X, C23 ou C24, que promete adicionar novas características à linguagem e substituir a C17, porém as notícias sobre essa versão são inconsistentes.

A linguagem C tem este nome pois foi inspirada na linguagem B, desenvolvida por Ken Thompson, anteriormente citado.

Ela eventualmente foi virar a “mãe” das linguagens de programação, inspirando a sintaxe de muitas linguagens, como C++, C#, Java, Javascript, PHP, além de outras.
