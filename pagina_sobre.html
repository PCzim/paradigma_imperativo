Paradigma Imperativo

Origem

O paradigma imperativo surgiu na década de 1940, quando os primeiros computadores digitais eletrônicos estavam sendo desenvolvidos. Esses computadores eram projetados com base na arquitetura de von Neumann, um modelo proposto por John von Neumann em 1945, que ainda influencia a maioria dos computadores modernos.

A arquitetura de von Neumann é baseada em um conceito simples: um computador deve ter uma unidade central de processamento (CPU), memória para armazenar tanto dados quanto instruções, e uma unidade de entrada e saída. Nesse modelo, o fluxo de execução é sequencial, e a CPU executa instruções uma de cada vez, alterando o estado da memória ao longo do tempo.

Principais Características da Linguagem Imperativa A programação imperativa é um paradigma de programação que descreve a computação como uma sequência de comandos que mudam o estado de um programa. Esse paradigma é frequentemente contrastado com a programação declarativa, que foca no que o programa deve realizar sem especificar como alcançar o resultado. Linguagens como Ada, ALGOL, Basic, C, PHP, Java, COBOL, Fortran, Pascal, Python, Lua e Mathematica seguem o modelo imperativo.

O conceito fundamental da programação imperativa é a Máquina de Turing, uma abstração matemática que representa funções computáveis. John von Neumann aprimorou esse conceito, criando uma arquitetura de computadores onde dados e programas são armazenados na mesma memória, e a CPU processa as instruções separadamente da memória. Essa arquitetura é a base dos computadores modernos.

Exemplo de Código:

#include <stdio.h>

int main() {
| for(int A = 0; A < 2; A++)
| | for(int B = 0; B < 2; B++)
| | | for(int C = 0; C < 2; C++)
| | | | printf("%d%d%d\n", A, B, C);
| return 0;
}
